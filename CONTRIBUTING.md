# Contributing to HBAR Wallet Desktop

Thank you for your interest in contributing! This document provides guidelines for contributing to the project.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/your-username/hbar-wallet-desktop.git`
3. Create a branch: `git checkout -b feature/your-feature-name`
4. Make your changes
5. Test thoroughly
6. Commit: `git commit -m "Add your feature"`
7. Push: `git push origin feature/your-feature-name`
8. Open a Pull Request

## Development Setup

See [INSTALL.md](INSTALL.md) for building from source.

## Code Style

- Follow C++17 standards
- Use consistent naming conventions
- Comment complex logic
- Keep functions focused and small

## Testing

```bash
cd build
ctest
```

## Pull Request Process

1. Update README.md if needed
2. Add tests for new features
3. Ensure all tests pass
4. Update documentation
5. Request review

## Security

Report security issues to: security@hbarwallet.com

Do NOT open public issues for security vulnerabilities.
